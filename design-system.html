Chapter One

The palette, and where each colour is allowed.

Every colour on the site is a token defined in the stylesheet. New colours are not added in a page; they are added to the tokens. The palette is intentionally narrow.

Do

  • Use maroon for anything the user must identify as institutional — CTAs, active nav, section rules.
  • Use rust for kickers and dates.
  • Use gold as a line, not a fill.
  • Alternate paper and paper-2 to give the page rhythm.

Don't

  • Never use Bootstrap blue (#0D6EFD) or any cold corporate primary.
  • Never introduce a purple, indigo, or teal gradient on white.
  • Never fill a large area with gold.
  • Never use rust for body text.
  • Never introduce a new brand colour without adding it to the tokens.
Chapter Two

Four families, and the discipline of how they meet.

The site pairs a heritage academic serif with a humanist grotesk, and mirrors both in Devanagari. No fifth family is introduced. Weights are limited. Italic is used with intent.

The families

  • EB Garamond — display and long-form. Weights 400 and 500; italic for emphasis.
  • IBM Plex Sans — UI, body, labels, navigation. Weights 300 to 600.
  • Noto Serif Devanagari — Hindi display and headings. Matches Garamond in role.
  • IBM Plex Sans Devanagari — Hindi body and UI. Matches Plex Sans in role.

All four are loaded from Google Fonts with display=swap. If Google Fonts must be self-hosted for policy reasons, the same four families in the same weights are the only substitution permitted.

Discipline

  • Headings set at weight 300 to 400 — the serif carries authority through its form, not its weight.
  • Letter-spacing on display is negative (-0.015em to -0.028em).
  • Kickers and labels are uppercase, tracked (letter-spacing 0.18em to 0.28em).
  • Italic is used in the serif only, for editorial emphasis — never in the sans.
  • Old-style numerals on display statistics; tabular numerals on tables and dates.

Do

  • Set every heading in EB Garamond (or Noto Serif Devanagari for Hindi).
  • Set every kicker as uppercase Plex Sans with 1px gold underline.
  • Use [lang="hi"] to switch families automatically.
  • Trust clamp(). Do not add a new fixed-size heading class.

Don't

  • Never introduce Inter, Roboto, Arial, system-ui, Space Grotesk, Lato, or Poppins.
  • Never set a heading in weight 700+.
  • Never set Hindi in the Latin serif or sans.
  • Never underline body text for emphasis.
Chapter Three

The grid, the rhythm, and the little details that carry it.

Everything sits inside one measured container. Sections alternate paper and paper-2. Rules are hair-thin. Maroon is used at 3px only where it must anchor.

The wrapper

  • Max width 1200px, centred.
  • Horizontal padding clamp(1rem, 2.5vw, 2rem).
  • Section vertical padding clamp(60px, 8vw, 96px) top and bottom.

Spacing scale

Multiples of 4px throughout. The scale in practice: 4, 8, 12, 16, 20, 22, 24, 28, 32, 40, 48, 56, 64, 80, 96. Do not invent intermediate values.

Rules and borders

  • Line (#D9D2C4) — 1px. Standard rule.
  • Line Soft (#E7E1D3) — 1px. Softer rule where standard is too heavy.
  • Maroon 3px — anchor rules under the primary nav, above the notices strip, at the start of a colleges directory.
  • Gold 1px — under kickers, under the active-nav underline, under dot indicators.

Shadows

Warm shadows only. The permitted shadow is drop-shadow(0 6px 16px rgba(42, 14, 12, 0.12)) — a soft maroon-tinted glow that reads as printed relief. Cool grey shadows are forbidden.

Paper grain

A double-layer radial-gradient noise sits behind the entire page at 50% opacity, using mix-blend-mode: multiply. Present but never announced. If a section feels flat, do not add gradients — check that the grain is rendering.

Borders and radii

  • Buttons: border-radius: 2px. Slight rounding — never pill buttons, never fully square.
  • Search chip: border-radius: 999px. The only pill in the system.
  • Cards: no radius. The editorial feel comes from sharp corners plus the rule border.
Chapter Four

The component library, one at a time.

The site is built from a fixed set of components. New pages are assembled from this set. A new component is added only when nothing in the set can be reasonably adapted.

Utility bar

Topmost strip. Maroon-ink background, gold-soft text. Accessibility controls (A+ / A−), quick links (Web Mail, Weather Advisory, Alumni, Success Stories), and the language toggle. Height 36px minimum, font-size 13px.

Masthead

Paper background, 1px line-bottom. The 72px crest sits left, bilingual wordmark centred against it, search chip and Admissions CTA on the right. On mobile the CTA collapses first; the search chip second.

Primary navigation

Sits under the masthead. 3px maroon bottom border anchors it. Active link marked in maroon with a gold 3px underline (width slightly less than the label, aligned left). Hover moves to maroon.

Hero carousel

Full-bleed image with a warm dark gradient overlay (rgba(20,8,7,0.85) to transparent). Bottom-anchored caption with a gold kicker (12px, tracked 0.24em) and a serif title at clamp(28px, 4vw, 52px). Arrows and dots only — no autoplay. Reduced motion respected.

Hero (non-carousel)

Asymmetric grid: title-and-lede on the left, crest plate with motto and metadata on the right, separated by a 1px vertical rule. Title at clamp(44px, 6.5vw, 82px), weight 300, letter-spacing -0.028em. Rust italic emphasis inside the title is the site's editorial signature.

Notices strip

Maroon-ink field. Italic serif label 'Notices' in gold-soft, gold count badge, five-column notice list, and a 'See all' link. Bottom bordered in 3px gold. Replaces the old ticker — no marquee.

Section heading

Every section opens with: a rust kicker (12px, uppercase, gold underlined), a serif H2 at clamp(30px, 3.4vw, 44px) in maroon-ink weight 350, and a short graphite sub-line. A 'See more' link may sit top-right of the heading block.

Statistic display

An oversized old-style numeral in the serif at clamp(130px, 18vw, 260px), paired with a short caption in the serif and a small dl of supporting figures on the right, separated by 1px line-soft rules.

Card — quick access

Paper background, 1px line-soft border, 3px maroon left border. Numbered kicker in gold at the top (e.g. '01'). Serif title, sans description, uppercase CTA line at the bottom over a hairline top rule. Hover translates 3px up and shifts the left border to gold.

College card / directory row

Numbered editorial directory — 22px serif number in gold on the left, 22px serif title in maroon-ink, one-line graphite description. 1px line-soft between rows, opened by a 2px maroon rule above. Hover slides the row 4px right.

News — editorial layout

One lead story on the left (16:10 image, kicker, date, serif headline, lede) and a numbered rail on the right (date, serif one-liner, hairline rules between items). No card grid. This is the site's distinctive news treatment.

Gallery card

Placed in the maroon-ink gallery section. Semi-transparent white fill, 1px semi-transparent border, gold kicker, white serif H3, muted body. Hover lifts the fill to 6% white and shifts the border to gold.

Trust / important links row

Grid of five equal cells with hairline dividers. On mobile collapses to three, then two columns while keeping the internal divider grid tidy.

Circulars / notices list

Two-column row: date on the left (140px, rust, uppercase, tabular), title on the right. Opened by 2px maroon rule. Hover shifts the title to maroon.

Footer

Maroon-ink field, gold 3px top border. Institutional wordmark, address block, contact block, compact link column. Gold-soft for section labels; paper-white for the wordmark.

Buttons

  • Primary CTA — background: maroon; color: white; padding: 11px 18px; border-radius: 2px. Hover to maroon-deep. An animated arrow (translate 3px on hover).
  • Ghost — transparent, maroon text, 1px line border. Hover to paper-2 fill and maroon border.
  • Search chip — the one pill in the system. 1px line border, transparent, 999px radius.

Skip link

Present on every page — a.skip with href #main. Hidden off-screen; on focus, appears at top-left on a maroon background. Non-negotiable for accessibility.

Component checklist for a new page

  • Skip link at the top
  • Utility bar
  • Masthead with crest and language toggle
  • Primary navigation with 3px maroon anchor
  • Section headings with kicker, H2, sub, optional see-more
  • Buttons only from the primary / ghost / search set
  • All content inside the 1200px wrap
  • Alternating paper / paper-2 backgrounds
  • Footer with maroon-ink field and gold top border
  • Every image has alt text
  • Every third-party image has referrerpolicy=no-referrer
  • No autoplay, no marquee, no infinite scroll
Chapter Five

Motion serves reading, never itself.

The site uses motion as punctuation. Every animated property has a defined role and a defined duration.

The permitted motions

  • Opacity crossfade — 900ms ease, for hero carousel slides.
  • Slow image zoom — 10s ease-out, transform scale 1.01 to 1.05, on active slide only.
  • Hover translate — 3px up for cards, 4px right for directory rows, 180ms ease.
  • Hover image zoom — scale 1.03 to 1.04, 500ms to 6s ease, on gallery and mosaic cards.
  • Colour transition — 180ms ease, on links, buttons, and border swaps.

Reduced motion

All non-essential motion is suspended under @media (prefers-reduced-motion: reduce). The hero carousel stops zooming; slide transitions become instantaneous.

Do

  • Use the durations above verbatim.
  • Honour prefers-reduced-motion on every new animation.
  • Prefer opacity and translate over complex keyframes.

Don't

  • Never autoplay carousels — user-driven navigation only.
  • Never introduce parallax scroll, mouse-tracking, or sticky video backgrounds.
  • Never animate text on entry.
  • Never use a marquee ticker.
Chapter Six

WCAG 2.1 AA, held as the floor.

The site is a public institutional face. Its accessibility target is WCAG 2.1 AA — not as an aspiration, as an obligation. Any regression against the checklist below is a bug, not a design decision.

Structure

  • Every page has one <main id="main"> landmark.
  • Every page has a skip link (a.skip) pointing at #main.
  • Landmarks used correctly — <header>, <nav aria-label>, <main>, <footer>.
  • Headings hierarchical. Exactly one H1 per page.

Keyboard

  • Every interactive element reachable and operable with the keyboard.
  • Focus rings visible — outline: 2px solid var(--gold); outline-offset: 2px.
  • Carousel arrows and dots focus-visible; escape key closes overlays.
  • No focus trap outside a modal.

Content

  • Every image has an alt. Decorative images use alt="".
  • Every form field has a real label. Placeholder is not a label.
  • Every icon-only button carries an aria-label.
  • Right-click, copy, and text selection are never blocked.

Colour and contrast

  • Body text meets 4.5:1. Large text meets 3:1. The pairings in Chapter One are the verified set.
  • Colour is never the sole carrier of meaning.

Motion

  • prefers-reduced-motion honoured on every animation.
  • Nothing autoplays audio. Nothing flashes more than three times per second.

Language

  • Document root sets lang="en" (or lang="hi").
  • Every Hindi run is marked lang="hi". CSS switches to Devanagari families automatically.
Accessibility fixes are never a phase two. Ship-blocking, always.
Chapter Seven

Bilingual — the DOM and CSS.

The philosophy of bilingual parity lives in the Design Philosophy document. This chapter records the technical implementation the team follows.

Marking Hindi in the DOM

Every Hindi run carries lang="hi". The CSS handles the rest — [lang="hi"] switches to the Devanagari sans; h1[lang="hi"], h2[lang="hi"], h3[lang="hi"], and the .brand-hi class switch to the Devanagari serif. Do not add Hindi text without the lang attribute — screen readers depend on it.

Typographic parity

  • Hindi headings are set in Noto Serif Devanagari at a size no more than one step below the Latin heading beside them.
  • Hindi body is set in IBM Plex Sans Devanagari, at 17px with slightly increased line-height (1.65 vs 1.55 for Latin).
  • Kickers in Hindi are set in the Devanagari sans, not the Latin.

Language toggle

  • Sits in the utility bar, always visible.
  • Two buttons — EN and हिन्दी. Active button marked in gold-soft with 1px gold underline.
  • Toggling switches the document's lang attribute and the language of content collections.

Do

  • Mark every Hindi run with lang="hi".
  • Set the Hindi wordmark at 15px in the masthead (matched to the Latin block).
  • Increase line-height for Devanagari.

Don't

  • Never render Hindi in the Latin serif or sans.
  • Never set Hindi smaller than the corresponding English on the same line.
  • Never machine-translate institutional content.
Chapter Eight

Logo — file, size, cross-origin.

The philosophy of the crest lives in the Design Philosophy document. This chapter records only the technical details the team needs.

Sizes

  • Masthead crest — 72 by 72 CSS pixels. Rendered from the optimised PNG (logo_small.png).
  • Hero crest plate — 190 CSS pixels wide, centred, with a soft drop shadow.
  • Favicon — the same PNG, served as rel="icon".

Format

The current asset is a PNG. Convert to SVG at the first opportunity — the crest is used at multiple sizes and the SVG will render crisply on high-DPI screens without shipping multiple bitmaps. The SVG must be reviewed against the master before it replaces the PNG.

Cross-origin images (referrer policy)

When the site pulls an image from a third-party host (for example, an officer photograph from a government page), add referrerpolicy="no-referrer" to the <img> tag. Some hosts block hot-linking when a referrer is sent; the no-referrer policy is what keeps those images resolving in Safari and iOS.

Security headers on the page

The document sets a Content Security Policy, a referrer policy, and X-Content-Type-Options: nosniff. These are defence-in-depth for a public institutional page. Do not remove them in new pages.

Chapter Nine

Governance of the codebase.

The philosophy of governance lives in the Design Philosophy document. This chapter records the code habits the team follows.

Tokens are edited in one place

All design tokens live in the :root block at the top of the site's stylesheet. Colour, font, spacing, rules — nothing is hand-coded elsewhere. If a new colour or size is needed, it is added to the tokens, then referenced. This is the single most important rule in the document.

Components are reused before invented

Before writing a new component, walk Chapter Four and ask whether an existing component can be adapted. A new component is added only when nothing fits. When one is added, it is added to Chapter Four in the next revision of this document.

New pages copy an existing page's skeleton

The homepage is the reference. Inner pages start from it and simplify.

Every change is checked against the checklist

Chapter Four closes with a checklist for a new page. Chapter Six sets the accessibility floor. Chapters One and Two set the token constraints. Before a change goes live, it is walked against these three lists.

Cache-busters on stylesheet and script references

The main index.html references style.css?v=N and script.js?v=N. Bump N whenever the stylesheet or script changes in a way readers must pick up on the next reload.
